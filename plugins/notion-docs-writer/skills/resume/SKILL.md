# 문서 작성 파이프라인 재개

이전에 시작한 문서 작성 파이프라인을 특정 단계부터 재개한다.
사용자가 중간 단계의 Notion 페이지를 수정한 뒤 이어서 진행하거나, 실패한 단계부터 다시 시작할 때 사용한다.

## 인자 파싱

사용자 입력에서 다음 두 가지를 파악한다:

1. **Notion 상위 페이지 URL** — step1~step5 하위 페이지들이 있는 상위 페이지
2. **시작 단계** — 어느 단계부터 재개할지 (1~6 또는 단계명)

매핑:
- `1` 또는 `draft` → Step 1 (개요 작성)부터
- `2` 또는 `research` → Step 2 (자료 수집)부터
- `3` 또는 `verify` → Step 3 (자료 검증)부터
- `4` 또는 `compose` → Step 4 (문서 작성)부터
- `5` 또는 `review` → Step 5 (문서 검토)부터
- `6` 또는 `plan` → Step 6 (업무 계획)부터

인자가 부족하면 물어본다:
- URL이 없으면: "Notion 상위 페이지 URL을 알려주세요."
- 단계가 없으면: "어느 단계부터 시작할까요? (1: 개요, 2: 자료수집, 3: 검증, 4: 문서작성, 5: 검토, 6: 업무계획)"

## 문서 유형 자동 판별

Notion 상위 페이지와 하위 페이지(STEP1_개요 등)의 내용을 읽어 문서 유형을 판별한다.

| 판별 기준 | 문서 유형 | 사용할 스킬 |
|---|---|---|
| "해결 방안", "핵심 문제 정의", "예상 리스크" 등의 섹션 | solve-problem | solve-problem 에이전트 세트 |
| (향후 추가될 문서 유형들) | ... | ... |

판별이 불확실한 경우 사용자에게 확인한다.

## 실행 전 검증

시작 단계에 필요한 이전 단계 결과가 존재하는지 Notion에서 확인한다:

| 시작 단계 | 필요한 하위 페이지 |
|---|---|
| Step 1 | 없음 (처음부터) |
| Step 2 | STEP1_개요 |
| Step 3 | STEP1_개요, STEP2_수집자료 |
| Step 4 | STEP1_개요, STEP3_자료검증 |
| Step 5 | STEP4_문서초안 |
| Step 6 | STEP5_최종결과 |

필요한 페이지가 없으면 사용자에게 알리고 어떻게 할지 확인한다:
- 이전 단계부터 다시 실행할지
- 그래도 현재 단계부터 시작할지 (누락된 입력 없이 진행 가능한 경우)

## 파이프라인 실행

검증이 통과되면, 판별된 문서 유형에 해당하는 에이전트 세트를 사용하여 지정된 단계부터 순차 실행한다.

각 단계의 에이전트 호출 방법은 해당 문서 유형 스킬의 Step 1~6 설명을 그대로 따른다.
Step 6 실행 시 work-planner에게 "상위 페이지 최상단에 TODO 목록을 삽입하라"고 지시한다.
차이점: 시작 단계 이전의 단계는 건너뛴다.

**주의**: 재실행하는 단계에 이미 하위 페이지가 존재하면, 에이전트에게 "기존 페이지를 덮어쓰라"고 지시한다.

## STEP 인터페이스 (공통)

| STEP | 에이전트 | 입력 | 출력 |
|------|---------|------|------|
| 1 | 유형별 전용 에이전트 | 유형별 상이 | `STEP1_개요` |
| 2 | `draft-researcher` (공용) | `STEP1_개요` | `STEP2_수집자료` |
| 3 | `research-verifier` (공용) | `STEP1_개요` + `STEP2_수집자료` | `STEP3_자료검증` |
| 4 | 유형별 전용 에이전트 | `STEP1_개요` + `STEP3_자료검증` | `STEP4_문서초안` |
| 5 | `document-reviewer` (공용) | `STEP4_문서초안` | `STEP5_최종결과` |
| 6 | `work-planner` (공용) | `STEP5_최종결과` | `STEP6_업무계획` + 상위 페이지 최상단 TODO 목록 |

## 완료 보고

해당 문서 유형 스킬과 동일한 형식으로 보고하되, 건너뛴 단계를 표시한다:

```
문서 작성 완료 (Step 3부터 재개)

최종 문서: [상위 페이지 URL]

각 단계별 기록:
- STEP1_개요: 개요 (기존 유지)
- STEP2_수집자료: 자료 수집 (기존 유지)
- STEP3_자료검증: 자료 검증 ← 재시작 지점
- STEP4_문서초안: 통합 문서
- STEP5_최종결과: 최종 검토본
- STEP6_업무계획: 업무 계획
```
