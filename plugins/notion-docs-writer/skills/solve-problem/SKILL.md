# 문제 해결 문서 작성 파이프라인

사용자의 문제를 분석하고, 체계적인 해결 방안 문서를 Notion에 작성하는 파이프라인을 실행한다.

## 실행 전 준비

1. 사용자에게 **문제 상황**을 확인한다.
   - 사용자가 이미 메시지에 문제를 설명했거나 Notion URL을 제공했으면 그대로 사용한다.
   - 정보가 부족하면 간단히 질문한다 (자세한 수집은 step1에서 에이전트가 처리).

2. Notion에 문서를 저장할 **상위 페이지**를 확인한다.
   - 사용자가 Notion URL을 제공했으면 그 페이지의 하위에 생성한다.
   - URL이 없으면 어디에 저장할지 물어본다.

3. 상위 페이지 URL을 확보하면 파이프라인을 시작한다.

## 파이프라인 실행

아래 6단계를 **순차적으로** 실행한다. 각 단계는 해당 에이전트를 Agent tool로 호출한다.
각 에이전트 호출 시 **상위 페이지 URL**과 **이전 단계의 결과물 위치**를 프롬프트에 포함한다.

### Step 1: 개요 작성 (solution-draft-writer)

Agent tool로 `solution-draft-writer` 에이전트를 실행한다.

프롬프트에 포함할 내용:
- 사용자가 제공한 문제 설명 전문
- Notion 상위 페이지 URL
- "이 페이지의 하위 페이지 'STEP1_개요'로 개요를 저장하라"는 지시

완료 확인: 에이전트가 STEP1_개요 페이지 URL을 반환하면 다음 단계로 진행한다.
사용자에게 "Step 1 완료: 개요 작성이 완료되었습니다." 와 함께 URL을 공유한다.

### Step 2: 자료 수집 (draft-researcher)

Agent tool로 `draft-researcher` 에이전트를 실행한다.

프롬프트에 포함할 내용:
- Notion 상위 페이지 URL
- "STEP1_개요 하위 페이지를 읽고, [조사 필요] 태그와 근거 없는 주장을 식별하여 자료를 수집하라"
- "결과를 'STEP2_수집자료' 하위 페이지로 저장하라"

완료 확인: STEP2_수집자료 페이지 URL을 받으면 다음 단계로 진행한다.
사용자에게 "Step 2 완료: 자료 수집이 완료되었습니다." 와 함께 URL을 공유한다.

### Step 3: 자료 검증 (research-verifier)

Agent tool로 `research-verifier` 에이전트를 실행한다.

프롬프트에 포함할 내용:
- Notion 상위 페이지 URL
- "STEP1_개요와 STEP2_수집자료 하위 페이지를 읽고, 수집된 자료의 정확성과 신뢰도를 독립적으로 검증하라"
- "결과를 'STEP3_자료검증' 하위 페이지로 저장하라"

완료 확인: STEP3_자료검증 페이지 URL을 받으면 다음 단계로 진행한다.
사용자에게 "Step 3 완료: 자료 검증이 완료되었습니다." 와 함께 URL을 공유한다.

### Step 4: 문서 작성 (document-composer)

Agent tool로 `document-composer` 에이전트를 실행한다.

프롬프트에 포함할 내용:
- Notion 상위 페이지 URL
- "STEP1_개요와 STEP3_자료검증을 읽고, 개요에 검증된 자료를 반영하여 완성 문서를 작성하라"
- "결과를 'STEP4_문서초안' 하위 페이지로 저장하라"

완료 확인: STEP4_문서초안 페이지 URL을 받으면 다음 단계로 진행한다.
사용자에게 "Step 4 완료: 문서 작성이 완료되었습니다." 와 함께 URL을 공유한다.

### Step 5: 문서 검토 (document-reviewer)

Agent tool로 `document-reviewer` 에이전트를 실행한다.

프롬프트에 포함할 내용:
- Notion 상위 페이지 URL
- "STEP4_문서초안 하위 페이지를 읽고, 독립적 관점에서 검증하여 직접 수정한 최종본을 생성하라"
- "결과를 'STEP5_최종결과' 하위 페이지로 저장하고, 상위 페이지 본문도 최종본으로 교체하라"

완료 확인: STEP5_최종결과 페이지 URL을 받으면 다음 단계로 진행한다.
사용자에게 "Step 5 완료: 문서 검토가 완료되었습니다." 와 함께 URL을 공유한다.

### Step 6: 업무 계획 (work-planner)

Agent tool로 `work-planner` 에이전트를 실행한다.

프롬프트에 포함할 내용:
- Notion 상위 페이지 URL
- "STEP5_최종결과 하위 페이지를 읽고, 문서 내용을 기반으로 우선순위별 업무 계획을 수립하라"
- "결과를 'STEP6_업무계획' 하위 페이지로 저장하라"
- "상위 페이지 최상단에 TODO 목록을 삽입하라"

완료 확인: STEP6_업무계획 페이지 URL을 받고, 상위 페이지 최상단에 TODO 목록이 삽입되면 파이프라인 완료.

## 파이프라인 완료 보고

모든 단계가 끝나면 사용자에게 최종 결과를 보고한다:

```
문서 작성 완료

최종 문서: [상위 페이지 URL]

각 단계별 기록:
- STEP1_개요: 개요
- STEP2_수집자료: 자료 수집
- STEP3_자료검증: 자료 검증
- STEP4_문서초안: 통합 문서
- STEP5_최종결과: 최종 검토본
- STEP6_업무계획: 업무 계획
```

## STEP 인터페이스

| STEP | 에이전트 | 입력 | 출력 |
|------|---------|------|------|
| 1 | `solution-draft-writer` | 사용자 문제 설명 + 상위 페이지 URL | `STEP1_개요`: 목차, 핵심 논점, `[조사 필요]` 태그 |
| 2 | `draft-researcher` (공용) | `STEP1_개요` | `STEP2_수집자료`: 조사 결과, 출처, 반영 제안 |
| 3 | `research-verifier` (공용) | `STEP1_개요` + `STEP2_수집자료` | `STEP3_자료검증`: 항목별 판정 (정확/부분수정/부정확/검증불가) |
| 4 | `document-composer` | `STEP1_개요` + `STEP3_자료검증` | `STEP4_문서초안`: 개요에 검증 자료 반영한 완성 문서 |
| 5 | `document-reviewer` (공용) | `STEP4_문서초안` | `STEP5_최종결과`: 최종 검토본 + 상위 페이지 교체 |
| 6 | `work-planner` (공용) | `STEP5_최종결과` | `STEP6_업무계획`: 우선순위별 업무 계획 + 상위 페이지 최상단 TODO 목록 |

## 에러 처리

- 특정 단계에서 에이전트가 실패하면 해당 단계에서 멈추고 사용자에게 알린다.
- 사용자가 문제를 해결한 뒤 `/notion-docs-writer:resume` 스킬로 해당 단계부터 재개할 수 있음을 안내한다.
- Notion 저장 실패 시 에이전트가 채팅으로 내용을 출력하므로, 사용자가 수동으로 저장할 수 있도록 안내한다.
