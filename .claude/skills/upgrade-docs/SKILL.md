# 기존 문서 업그레이드 파이프라인

기존 Notion 문서를 분석·검증하여 업그레이드하는 파이프라인을 실행한다.

## 실행 전 준비

1. 사용자에게 **업그레이드할 Notion 문서 URL**을 확인한다.
   - 사용자가 이미 URL을 제공했으면 그대로 사용한다.
   - URL이 없으면 물어본다.

2. Notion 문서를 읽어 **문서 유형을 판별**한다.
   - "해결 방안", "핵심 문제 정의", "예상 리스크" 등 → solve-problem
   - (향후 추가될 문서 유형들)
   - 판별이 불확실하면 사용자에게 확인한다.

3. 유형이 확정되면 파이프라인을 시작한다.

## 파이프라인 실행

아래 6단계를 **순차적으로** 실행한다.
Step 1은 draft-rewriter 에이전트를 사용하고, Step 2~6은 판별된 문서 유형의 에이전트를 사용한다.

### Step 1: 개선 개요 작성 (draft-rewriter)

Agent tool로 `draft-rewriter` 에이전트를 실행한다.

프롬프트에 포함할 내용:
- 업그레이드할 Notion 문서 URL
- "기존 문서를 읽고 구조/논리/근거/표현을 분석하여 개선된 개요를 작성하라"
- "원본을 'STEP0_원본백업'으로 백업하고, 개선 개요를 'STEP1_개요'로 저장하라"

완료 확인: 에이전트가 STEP1_개요 페이지 URL을 반환하면 다음 단계로 진행한다.
사용자에게 "Step 1 완료: 개선 개요 작성이 완료되었습니다." 와 함께 URL을 공유한다.

### Step 2~6: 판별된 문서 유형의 파이프라인 실행

판별된 문서 유형(예: solve-problem)의 Step 2~6을 그대로 따른다.

- Step 2: 자료 수집 (draft-researcher)
- Step 3: 자료 검증 (research-verifier)
- Step 4: 문서 작성 (document-composer)
- Step 5: 문서 검토 (document-reviewer)
- Step 6: 업무 계획 (work-planner)

각 단계의 에이전트 호출 방법은 해당 문서 유형 스킬(예: `/notion-docs-writer:solve-problem`)의 설명을 참조한다.

## 파이프라인 완료 보고

모든 단계가 끝나면 사용자에게 최종 결과를 보고한다:

```
문서 업그레이드 완료

최종 문서: [상위 페이지 URL]
문서 유형: [판별된 유형]

각 단계별 기록:
- STEP0_원본백업: 원본 백업
- STEP1_개요: 개선 개요
- STEP2_수집자료: 자료 수집
- STEP3_자료검증: 자료 검증
- STEP4_문서초안: 통합 문서
- STEP5_최종결과: 최종 검토본
- STEP6_업무계획: 업무 계획
```

## STEP 인터페이스

| STEP | 에이전트 | 입력 | 출력 |
|------|---------|------|------|
| 0 | `draft-rewriter` 내부 처리 | 기존 문서 | `STEP0_원본백업`: 원본 그대로 + 백업 일시 |
| 1 | `draft-rewriter` | 기존 문서 URL | `STEP1_개요`: 개선된 구조/논점, `[조사 필요]` 태그 |
| 2 | `draft-researcher` (공용) | `STEP1_개요` | `STEP2_수집자료`: 조사 결과, 출처, 반영 제안 |
| 3 | `research-verifier` (공용) | `STEP1_개요` + `STEP2_수집자료` | `STEP3_자료검증`: 항목별 판정 |
| 4 | 판별된 유형의 composer | `STEP1_개요` + `STEP3_자료검증` | `STEP4_문서초안`: 완성 문서 |
| 5 | `document-reviewer` (공용) | `STEP4_문서초안` | `STEP5_최종결과`: 최종 검토본 + 상위 페이지 교체 |
| 6 | `work-planner` (공용) | `STEP5_최종결과` | `STEP6_업무계획`: 우선순위별 업무 계획 |

## 에러 처리

- 특정 단계에서 에이전트가 실패하면 해당 단계에서 멈추고 사용자에게 알린다.
- 사용자가 문제를 해결한 뒤 `/notion-docs-writer:resume` 스킬로 해당 단계부터 재개할 수 있음을 안내한다.
- Notion 저장 실패 시 에이전트가 채팅으로 내용을 출력하므로, 사용자가 수동으로 저장할 수 있도록 안내한다.
